## 🔴 Фаза 1 — Критичне (фінансовий облік + одиниці)

### Крок 1.1 — Ціна в залишках (середньозважена)

**Що зробити:**
- Додати `stock_balances.unit_price NUMERIC(12,2)`
- Додати `stock_movements.unit_price NUMERIC(12,2)`
- Оновити `update_stock_balance` — приймати ціну, перераховувати середньозважену
- Оновити `confirm_receipt` — передавати ціну з `receipt_items.price`
- Оновити дашборд (stock_value через unit_price а не purchase_price)
- Міграція SQL + RPC

**Тест:** оприбуткувати товар за ціною 100 → залишок 10 шт × 100 = 1000 грн → оприбуткувати ще 5 шт по 120 → середня = 106.67 → залишок = 15 × 106.67 = 1600

### Крок 1.2 — Довідник одиниць виміру + конвертація

**Що зробити:**
- Створити таблицю `units` (id, code, name, category)
- Створити таблицю `unit_conversions` (from_unit_id, to_unit_id, factor)
- Додати `products.base_unit_id`, `products.purchase_unit_id`
- Заповнити базові одиниці (шт, кг, л, уп, пач, ящ, м, рул)
- Оновити Telegram-бот — при показі кількості конвертувати в базові одиниці
- Міграція SQL + оновлення code

**Тест:** товар "Пакет для сміття 120л" — базова одиниця "шт", закупівельна "уп" (1 уп = 5 шт) → замовлення "2 уп" → в заявці 10 шт
